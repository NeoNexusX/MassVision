环境搭建：

```bash
cd MassVision
npm install
# 可选：如果项目使用到 crypto-js (见 src/utils/auth.js)，请安装：
npm install crypto-js --save
npm run format
npm run dev
```

项目结构（简要）：

- 根目录：`package.json`（依赖与脚本）、Vite/TS 配置、测试配置等。
- `public/`：静态资源（不经构建处理）。
- `src/`：源码
   - `main.ts`：应用入口，初始化 `Pinia` 与 `vue-router`。
   - `router/`：路由定义（使用 `createWebHistory`）。
   - `stores/`：Pinia store（组合式 API）。
   - `components/`, `views/`, `assets/`：常规组件/页面/样式资源。

关键依赖（从 package.json 与源码确认）：

- 框架与构建：
   - **Vue**: ^3.5.25
   - **Vite**: ^7.2.4
   - **TypeScript**: ~5.9.0
   - **@vitejs/plugin-vue** / **@vitejs/plugin-vue-jsx**（用于 Vue 单文件组件与 JSX 支持）

- 状态与路由：
   - **Pinia**: ^3.0.4
   - **vue-router**: ^4.6.3

- 网络与样式：
   - **axios**: ^1.13.2（项目在 `src/utils/http.js` 中创建了 axios 实例，使用 `import.meta.env.VITE_API_BASE` 作为 baseURL）
   - **tailwindcss**: ^4.1.18
   - **daisyui**: ^5.5.14

- 测试与工具链：
   - **Vitest**: ^4.0.14（单元测试）
   - **Playwright**: ^1.57.0（E2E）
   - **vue-tsc**（类型检查）、**eslint**、**prettier**

注意与实现细节（基于源码）：

- `src/main.ts`：挂载 `Pinia` 与 `router`，说明使用组合式 API 与插件式注册。
- `src/router/index.ts`：路由使用 `createWebHistory`，并且有注释掉的路由守卫示例（基于 localStorage 的 jwtToken）；默认根路径重定向到 `/login`。
- `src/utils/http.js`：创建了 axios 实例，baseURL 读取 `VITE_API_BASE` 环境变量，拦截器部分被注释，可按需启用并将 token 添加到请求头。
- `src/utils/auth.js`：使用 `crypto-js` 对密码做 SHA256（文件中引入了 `CryptoJS`，但 `crypto-js` 未必列在 package.json 中，若使用请安装）。
- `src/stores`：使用 Pinia 的组合式 `defineStore`（示例：`counter` store）。

常用脚本（见 `package.json`）：

- `npm run dev`：启动开发服务器（Vite）
- `npm run build` / `npm run build-only`：构建生产包
- `npm run preview`：预览构建产物
- `npm run type-check`：运行 `vue-tsc` 做类型检查
- `npm run lint`：运行 ESLint（含 --fix）
- `npm run format`：运行 Prettier 格式化 `src/`（项目目前在 scripts 中为 `prettier --write --experimental-cli src/`）
- `npm run test:unit`：运行单元测试（Vitest）
- `npm run test:e2e`：运行 E2E（Playwright）

建议：

- 开发流程：先运行 `npm install`，再 `npm run dev`。在提交前先运行 `npm run format` 与 `npm run lint`。
- 若开启后端通信并使用 token，请在 `src/utils/http.js` 中恢复并完善请求拦截器，将 `Authorization` 头注入请求。
- 如果使用 `src/utils/auth.js` 中的 SHA256 功能，请安装 `crypto-js`：`npm install crypto-js --save`。
- 若需在生产中配置 API 地址，请通过 `.env` 或 CI/CD 注入 `VITE_API_BASE`。

- 或运行一次本地 `npm run dev` 以验证启动流程（需在你的机器上运行）。
